Задание 3. Выполнено. Описание задачи и её решения ниже.
Подумать над глобальными сущностями. К примеру, сущность «Продукт» в интернет-магазине актуальна не только для корзины, но и для каталога. 
Стремиться нужно к тому, чтобы объект «Продукт» имел единую структуру для различных модулей сайта, но в разных местах давал возможность вызывать 
разные методы.
Решение.
Продумал несколько основных глобальных сущностей для интернет-магазина:
1. Пользователь.
Данная сущность должна иметь следующие атрибуты: логин, пароль, имя, фамилия, адрес(а) доставки, дата рождения, телефон, почта.
2. Продукт.
Данная сущность должна иметь следующие атрибуты: наименование, цена, описание, характеристики, категория.
3. Каталог.
Данная сущность должна включать следующие атрибуты: массив продуктов.
Данная сущность должна иметь следующие методы: поиск продукта(ов) по характеристикам, отображение всех продуктов, отображение продуктов по категориям.
4. Корзина.
Данная сущность должна включать следующие атрибуты: массив продуктов в корзине.
Данная сущность должна иметь следующие методы: добавить продукт в корзину, удалить продукт из корзины, посчитать цену корзины.
5. Заказ.
Данная сущность должна включать следующие атрибуты: массив продуктов для заказа.
Данная сущность должна иметь следующие методы: отобразить заказ, изменить заказ, оформить заказ.
6. Склад.
Данная сущность должна включать следующие атрибуты: массив продуктов на складе.
Данная сущность должна виметь следующие методы: добавить продукт(ы) на склад, изъять продукт(ы) со склада, отобразить продукты на складе.
7. Производитель.
Данная сущность должна включать следующие атрибуты: наименование, юридический адрес, почтовый адрес, ИНН, КПП, банковские реквизиты, 
телефон, электронная почта, генеральный директор (объект Пользователь).
8. Комментарий к продукту.
Данная сущность должна включать следующие атрибуты: пользователь (объект), написавший комментарий, текст комментария.
